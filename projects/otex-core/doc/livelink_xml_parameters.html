Configuration Parameters in the XML Configuration File

Name Description Values and Usage
HTTP Tunneling Parameters
caRootCerts The root certificates to use when verifying the Web server's certificate. This parameter is used only if HTTP tunneling is enabled (through the livelinkCgi parameter), and the https and verifyServer parameters are both set to true. The default value is specified through the GSA Admin Console. Additional root certificates can be specified here.
Separate Authentication Parameters
authenticationCaRootCerts The root certificates to use when verifying the Web server's certificate. This parameter is used only if HTTP tunneling is enabled (through the authenticationLivelinkCgi parameter), and the authenticationHttps and authenticationVerifyServer parameters are both set to true. The default value is specified through the GSA Admin Console. Additional root certificates can be specified here.
Indexing and Searching Parameters
servtype The database server type. Valid values begin with "MSSQL" or "Oracle". Sybase is not supported. This value may be copied from the servtype parameter in the [dbconnection:connection_name] section of the opentext.ini file. The default value is determined automatically.
startDate The repository is traversed in modification date order, beginning with the earliest item. When a start date is specified, the indexing begins with items modified on or after the start date. Items with an earlier modification date will not be indexed. A date and time specified in the default formatting style or the short formatting style for the default locale. The default value is empty.
publicContentUsername Indexed items that are accessible by this user are marked as public documents. Public documents do not require authentication or authorization.

Note: This parameter bypasses Livelink security. Items that are made public are shown in the search results without authentication or authorization. The content of these items is available through the "Cached" and "Text Version" links. These items are not accessible within Livelink unless you have a Livelink customization that makes this possible.

 A Livelink username. The default value is empty.
excludedNodeTypes Specifies the node types that you want to exclude from indexing. Excluding a node type means that no items of that type are indexed. A comma-separated list of subtype numbers, optionally enclosed by braces. This value may be copied from the ExcludedNodeTypes parameter in the [LivelinkExtractor] section of the opentext.ini file. The default value is "137,142,143,148,150,154,161,162,201,203,209,210,211".
excludedVolumeTypes Specifies the volume types that you want to exclude from indexing. Excluding a volume type means that no volumes of that type or any of the items that the volumes contain are indexed. A comma-separated list of volume type numbers, optionally enclosed by braces. If this parameter is not specified, no volume types are excluded from indexing. This value may be copied from the ExcludedVolumeTypes parameter in the [LivelinkExtractor] section of the opentext.ini file. The default value is "148,162".
excludedLocationNodes Specifies the IDs of the nodes that you want to exclude from indexing. Excluding a node means that it is not indexed, and if it is a container, none of the items it contains are indexed. A comma-separated list of object IDs, optionally enclosed by braces. The default value is empty.
displayPatterns A map of the relative display URLs used in search results. Each subtype can have a different relative URL. This URL suffix is combined with the value of the displayUrl parameter to form a complete URL for each item in the search results. The map contains keys that consist of comma-separated subtype integers, or the special string "default". These are mapped to a java.text.MessageFormat pattern. There are four parameters that can be used in each pattern, delimited by braces:
0
The object ID.
1
The volume ID.
2
The subtype.
3
The display action, which varies by subtype and is configured by the displayActions parameter.
The default value is a map with one entry mapping "default" to "?func=ll&objAction={0}&objId={3}".
displayActions A map of the display actions used in search results. Each subtype can have a different action. The map contains keys that consist of comma-separated subtype integers, or the special string "default". These are mapped to Livelink action names, such as "browse" or "overview". If the map contains an entry mapping "144" (documents) to "overview", and the Livelink server is version 9.2 SP1 or earlier, the "properties" action is used instead. The default value is a map with one entry mapping "default" to "properties".
contentHandler The name of the ContentHandler implementation class to use when retrieving document content for indexing. There are four ContentHandler implementations:
ByteArrayContentHandler
The fastest implementation, but not a very scalable one because it maintains the entire content in a byte array.
FileContentHandler
Stores the content in a temporary file, but performs very badly for files larger than 10 MB.
HttpURLContentHandler
Retrieves the content directly from the Livelink server using an HttpURLConnection. Performance is inexplicably disappointing, in line with but consistently 1-1/2 to 3 times slower than the others.
PipedContentHandler
Uses a pipe to match the in-memory performance characteristics of the byte array handler without the scalability issues. Combines consistently good performance with small memory consumption.
The default value is "com.google.enterprise.connector.otex.FileContentHandler"
includedExtendedData A map of the ExtendedData attributes that are indexed and used to construct the HTML content. Each subtype can have different attributes indexed. The map contains keys that consist of comma-separated subtype integers. The special string "default" is not supported. These are mapped to a comma-separate list of attribute names, which should appear in the ExtendedData field of the given subtypes. The default value is an empty map.
Unimplemented Parameters
maxItemsPerExtraction The largest batch size that is used. If the batch size hint is larger than this, the maximum size is used instead.

This value may be copied from the MaxItemsPerExtraction parameter in the [LivelinkExtractor] section of the opentext.ini file.

 The value must be between 1 and 100000. The default value is 5000.
showHiddenItems Specifies whether hidden items are indexed and searchable. One of the following values:
{} or false
Do not index hidden items.
{'ALL'} or true
Index all hidden items.
A comma-separated list of subtype numbers, optionally enclosed by braces
Index hidden items within the container types specified.

This value may be copied from the ShowHiddenItems parameter in the [Atlas] section or in the [Explorer] section of the opentext.ini file. The default value is "{}".
includedCategories The IDs of the categories that you want to index. Including a category means that when that category is applied to an item, the attributes are indexed as metadata for that item. A comma-separated list of object IDs for categories, and/or one of the following special strings:
searchable
Only attributes that have the Show in Search parameter enabled, from each of the categories that are applied to the item, are indexed. Otherwise, all attributes of the Categories are indexed.
all
All attributes from each of the categories that are applied to the item are indexed.
none
No attributes from any of the categories that are applied to the item are indexed. This value turns off the indexing of categories.

Specifying an empty string is the same as "all." The default value is "all,searchable".
excludedCategories The IDs of the categories that you do not want to index. Excluding a category means that when that category is applied to an item, the attributes are not indexed as metadata for that item. A comma-separated list of object IDs for categories, or one of the following special strings:
all
All attributes from each of the categories that are applied to the item are excluded. This value turns off the indexing of categories.
none
No attributes from any of the categories that are applied to the item are excluded.

Specifying an empty string is the same as "none". The default value is empty.
includedObjectInfo The object info attributes that you want to index. A comma-separated list of attribute names. For a list of object info attributes, see "ObjectInfo Attributes" in the Livelink API Developer's Reference Guide from Open Text. The default value is "Comment,CreatedBy,CreatedDate,ID,ModifyDate,Name,SubType,UserID".
includedVersionInfo The version info attributes that you want to index. A comma-separated list of attribute names. For a list of version info attributes, see "VersionInfo Attributes" in the Livelink API Developer's Reference Guide from Open Text. The default value is "MimeType".
includedSystemAttributes The system attributes, or additional node attributes, that you want to index. A comma-separated list of attribute names. The default value is empty.
